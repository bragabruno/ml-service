# ADR-002: dbt Dual-Target (DuckDB + Postgres)

**Status:** Accepted  
**Date:** 2026-06-02

## Context

dbt models need to run in two contexts: local development (zero-infra, fast iteration) and production parity (Postgres, matching the platform's actual database).

## Decision

Configure dbt with two targets sharing the same models:
- **duckdb** (default): Reads directly from the local DuckDB file generated by `data/generate_synthetic.py`. Zero setup.
- **postgres**: Connects to the docker-compose Postgres instance. Used for CI parity and production.

Models use only SQL that compiles on both adapters (no Postgres-specific extensions).

## Consequences

- Developers run `dbt build --target duckdb` with no infrastructure
- CI validates both targets compile
- Same models deploy to production without modification
- Trade-off: some Postgres-specific optimizations are deferred
